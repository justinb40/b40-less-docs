Responsive Navigation Elements component

In addition to the page component, there is an assortment of page-nav classes that work together to provide tools to build mobile and responsive navigation.

JavaScript Dependency

The responsive page-nav elements rely on additional logic to handle revealing and dismissing the navigation panels. Include these scripts at the bottom of your document before the closing </body> tag.


<script src="public/scripts/less/less.common.min.js"></script>
<script src="public/scripts/less/less.nav-toggle.min.js"></script>

Classes

Class Usage
fixed-when-active Set the affected element to be fixed to the top of the viewport when the nav-is-active class is applied to the body. Typically applied to the page-nav element.
page-nav-body Container wrapper for the navigation menu and/or other visible content that is toggled.
page-nav-content The scrollable portion of the navigation menu. This class exists separately to make development and debugging easier and must be included immediately inside of page-nav-body.
page-nav-fade Modifier class for making the page-nav-body element fade in.
page-nav-header While somewhat optional, this element is fixed to the top of the page-nav element when the nav-is-active class is applied to the body. Typically contains the page logo and the navicon element.
page-nav-left Float this element to the left for breakpoints at or ABOVE the navigation breakpoint. Typically applied to the page-nav-header element.
page-nav-right Float this element to the right for breakpoints BELOW the navigation breakpoint. Typically applied to the page-nav-body element.
page-nav-slide-down Modifier class for making the page-nav-body element slide in from the top of the viewport. (See source code for a description of the flaws with this class)
page-nav-slide-left Modifier class for making the page-nav-body element slide in from the right side of the viewport.
page-nav-slide-right Modifier class for making the page-nav-body element slide in from the left side of the viewport.
page-nav-slide-up Modifier class for making the page-nav-body element slide in from the bottom of the viewport.
page-nav-spacer Because the nav-is-active state will cause a few elements to be pulled from the DOM and fixed to the viewport, the page will appear to "jump up". This convenience class that will apply the same value of @page-nav-header-height to the height of an element to reserve the space once occupied by those fixed elements. If the height applied is not sufficient, then simply override the values in the theme file with your own implementation. Typically applied to the page-header element.
page-nav-wrapper Handle container and container-fluid elements by removing their padding and max-width below the navigation breakpoint.

Usage

<header class="page-header page-nav-wrapper page-nav-spacer">
    <nav class="page-nav fixed-when-active">
        <div class="container">
            
            <div class="page-nav-header page-nav-left">
                <h2 class="logo">
                    <a href="/">
                        <img src="/sites/back40design/templates/default/images/logo.jpg" alt="Back40 Design">
                    </a>
                </h2>
                
                <button type="button" class="navicon navicon-dismissible" data-toggle-nav="#nav-menu">
                    <span class="sr">Toggle Navigation</span>
                    <span class="navicon-bar"></span>
                    <span class="navicon-bar"></span>
                    <span class="navicon-bar"></span>
                </button> 
            </div>
            
            <div class="page-nav-body page-nav-right page-nav-slide-left page-nav-fade" id="nav-menu">
                <div class="page-nav-content">
                    <!nav id="nav" type="nested" /!>
                </div>
            </div>
            
        </div>
    </nav>
</header>

JavaScript Events

Event Usage
b40.nav.toggled Broadcast when the navigation is toggled.

Using Events

// Bind to the `body` to listen for navigation events
$('body').bind('b40.nav.toggled', function(event, data) {
    // `data.isActive` is a boolean indicating whether or not the navigation is currently active
    console.log('Navigation active state: ' + `data.isActive);
});
