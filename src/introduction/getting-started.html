Getting Started

Project Files

The template directory structure of projects will typically follow the example below:

default/
├── css/
│   ├── _less/
│   │  ├── compiler.less
│   │  ├── config.less
│   │  └── theme.less
│   └── style.min.css
├── icons/
│   ├── favicon.ico
│   ├── apple-touch-icon-57x57.png
│   ├── apple-touch-icon-72x72.png
│   ├── apple-touch-icon-114x114.png
│   └── apple-touch-icon-144x144.png
├── images/
├── js/
│   ├── _src/
│   │  ├── common.js
│   │  └── index.js
│   ├── common.min.js
│   └── index.min.js
├── index.html
└── inside.html
  • compiler.less imports resources and dictates load order.
  • config.less defines variables and mixins and overrides defaults.
  • theme.less defines the custom styles for the project.
  • style.min.css contains the final, minified output from compiler.less.

Device Icons

For mobile and responsive websites, additional device icons need to be created and stored in the icons/ directory.

Javascript Files

You may have some javascript files to include with your project. Source versions of your javascript files should be stored in the js/_src/ directory. Minified versions of these files are stored directly in the js/ directory and have .min appended to the file name. To minify your javascript files, use javascript minifier.

See documentation on using scripts for more information.
