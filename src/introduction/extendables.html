Extendables

These classes are not actually defined as mixins. They are imported by (reference) and are meant to be extended rather than called as a mixin.

Extend vs. Mixin

Calling a class as a mixin will create a new instance of the rules within it. However, extending the class adds the selector to an ongoing list of selectors and applies the rules from the mixin only one time, thus significantly reducing code bloat.

Anti-pattern

.block1 {
    .border-box-sizing();
}
.block2 {
    .border-box-sizing();
}
.block3 {
    .border-box-sizing();
}


/* will output... */
.block1 {
    -webkit-box-sizing: border-box;
    -moz-box-sizing: border-box;
    box-sizing: border-box;
}
.block2 {
    -webkit-box-sizing: border-box;
    -moz-box-sizing: border-box;
    box-sizing: border-box;
}
.block3 {
    -webkit-box-sizing: border-box;
    -moz-box-sizing: border-box;
    box-sizing: border-box;
}

Preferred pattern

.block1 {
    &:extend(.border-box-sizing all);
}
.block2 {
    &:extend(.border-box-sizing all);
}
.block3 {
    &:extend(.border-box-sizing all);
}


/* will output... */
.block1,
.block2,
.block3 {
    -webkit-box-sizing: border-box;
    -moz-box-sizing: border-box;
    box-sizing: border-box;
}
