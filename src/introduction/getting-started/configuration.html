Configuration

Variables from the framework's core are overridden in config.less. The @javelin-sitename var is also established in the same file as well as any custom variables or mixins created specially for the project.

Defining custom variables and mixins

Both are meant for a high amount of reusability and while not forbidden, neither should be too common in your work. For example, if you are using a custom variable only a couple of times, then it might not be necessary at all. Just remember that you're trying to introduce convenience and save time.

Some possible use cases for creating custom variables or mixins are:

  • Reusing dominant colors from the theme.
  • Referencing height values.
  • Projects that are duplicated and change the color theme.

Setting Main Background Colors

Don't forget to override the @html-bg and @body-bg variables. Typically, the @html-bg color will be set to the bottom-most, dominant pixel color in the footer.

Overriding Generic Base Values

Several of the default values for core variables are referencing a generic base value such as borders, padding, media queries or font families. These base values are there to expedite the styling process and enforce consistency across the styleguide. For example, changing any of the padding base values may cause drastic changes so make sure you know what you are affecting before changing these values.
