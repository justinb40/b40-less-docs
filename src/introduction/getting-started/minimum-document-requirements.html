Minimum Document Requirements

HTML5 Standards

HTML documents should conform to HTML5 standards. Some main points to cover are:

  • Doctype should be set to <!DOCTYPE html>.
  • Specify character encoding with <meta charset="utf-8">.
  • Include <meta http-equiv="X-UA-Compatible" content="IE=edge"> in the document head to make IE render using the latest version on the machine and avoid the dreaded "compatibility mode".
  • Avoid self closing slashes (i.e. <img src="" />) for elements like <br>, <hr>, <img>, <input> and <meta>. HTML5 does not require them.
  • Avoid specifying the type attribute for <script> and <style> elements. HTML5 will assume type="text/javascript" and type="text/css" by default.

Responsive and Mobile Requirements

When you are developing responsive or mobile projects, you will need to set either @is-responsive or @is-mobile to true and add this <meta> tag to the <head>:

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Supporting IE8 (in a lazy, but good enough way)

While we don't make a strong point to support IE8 anymore, its still easy to make them feel somewhat special by including the following script inside of an IE conditional tag. Any version of IE that came before IE9 will gain polyfills such as CSS3 selectors, media query breakpoints and additional polyfills for HTML5 and ECMAScript 5 usage.

<!--[if lt IE 9]>
<script src="public/scripts/ie8-polyfills.min.js"></script>
<![endif]-->
