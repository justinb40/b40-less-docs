Navicon component

Create a pure-css <button> toggle for off-canvas navigation that is positioned to the top right of its container. Alternatively, the navicon can be positioned to the top left by applying left: @navicon-offset-right; right: auto; to the element.

JavaScript Dependency

Using with the Canvas Panels component relies on additional logic to handle revealing and dismissing the canvas panels. Include these scripts at the bottom of your document before the closing </body> tag.

Optionally, include hammer.min.js to allow the user to dismiss a side panel by swiping.


<script src="public/scripts/hammer.min.js"></script>
<script src="public/scripts/less/less.common.min.js"></script>
<script src="public/scripts/less/less.canvas-panels.min.js"></script>

Classes

Class Usage
navicon Container class typically applied to a <button> element.
navicon-bar Apply to three <span> elements to create the stack of horizontal lines which act as a visual cue for a sliding menu.
navicon-dismissible Apply to the <button> element to convert a group of navicon bars into a "close" icon when the navigation is active. This class assumes that 3 navicon-bar elements are present.

Usage

<button type="button" class="navicon navicon-dismissible open-right-canvas">
    <span class="sr">Toggle Navigation</span>
    <span class="navicon-bar"></span>
    <span class="navicon-bar"></span>
    <span class="navicon-bar"></span>
</button>
