Dropdown component

Create custom dropdown menus similar to the HTML <select> element.

JavaScript Dependency

This component relies on additional logic to handle toggling the dropdown menu, broadcasting dropdown events and storing the dropdown value. Include these scripts at the bottom of your document before the closing </body> tag.


<script src="public/scripts/less/less.common.min.js"></script>
<script src="public/scripts/less/less.dropdown.min.js"></script>

Avoid using as a standard form component

Using this component in web forms is not suggested as it does not offer the full set of usability features, accessibility features and testing background that comes with the Native <select> element. Classes

Class Usage
dropdown Container for the dropdown.
dropdown-label The always-visible part of the dropdown that holds the current selected option's label value.
dropdown-label-text The wrapper class for the text portion of the .dropdown-label.
dropdown-caret The arrow within .dropdown-label used to indicate dropdown functionality.
dropdown-options The menu that drops down when the user activates the dropdown.
dropdown-option A clickable <li> element used in place of the standard <option> element. Use the data-value attribute to store the option's value instead of the standard value attribute.
dropdown-slug A clickable option that is used as an empty option suggesting no input. Clicking this option unsets the value of the dropdown back to undefined.
dropdown-group A non-clickable <li> element used to add group labels between dropdown options. This element should be a sibling of .dropdown-option. It does not wrap or contain any other elements.

JavaScript Attributes

Attribute Usage
data-target Set the value of this attribute on the .dropdown container element to the ID of the hidden input nested within the dropdown as well as the value for the for attribute on the .dropdown-label element. Be sure to include the hash # in the data-target value.
data-value Set the value of this attribute on the .dropdown-option element to store the option's value. Empty strings are allowed but the absence of this attribute on an option will have a value of undefined. Always omit this attribute on the .dropdown-slug element.

Class Methods

These methods are available on the event.dropdown object passed into event callbacks. This is an actual instance of the dropdown class and it has a list of methods you can use in your own programming.

Method Usage
deselect() Deselect any selected options and set the dropdown value to undefined.
get() Return the jQuery extended element associated with the dropdown.
getSelected() Return the selected option as a data object containing the element, label and value.
hide() Hide the dropdown menu.
setByOption(target) Set the selected option by providing the target element that would normally be clicked by the user in the dropdown menu.
setByValue(value) Set the selected option by providing the value that matches an option in the dropdown menu. If there are multiple options with the same value, the first option found with that value will be used.
show() Show the dropdown menu.
toggle() Toggle the dropdown menu.
val() Return the dropdown's current value.

JavaScript Events

Event Usage
b40.dropdown.change Event triggered when the value of the dropdown changes to a different value.
b40.dropdown.hide Event triggered when the dropdown menu is dismissed.
b40.dropdown.show Event triggered when the dropdown menu is revealed.

Using Events

$('.dropdown').bind('b40.dropdown.change', function(event) {
    // The getSelected() method returns an object with 3 properties...
    var data = event.dropdown.getSelected();
    
    // ...the label and value for the selected option
    console.log(data.label + ': ' + data.value);
    
    // ...and a reference to the element associated with the selected option
    console.log(data.target);
});
